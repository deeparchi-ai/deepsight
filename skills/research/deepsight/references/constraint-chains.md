# DeepSight 约束追问链

> 从 `SKILL.md` Step 2a 拆分。按问题类型选择对应约束链，每链多轮递进追问。

## 默认三维约束链（工程/材料/技术类）

```
Round 0: 无约束自由输出 → 基线回答（技术路线、数据罗列）
Round 1: 物理极限 — "离理论上限多远？损耗能分解吗？"
Round 2: 工程放大 — "从实验室到量产，瓶颈在哪个参数？"
Round 3: 成本结构 — "Fermi 估算：每单位成本数量级？"
```

## 金融/投资/商业分析类

```
Round 0: 无约束自由输出 → 基线分析
Round 1: 护城河检验 — "这个优势能持续多久？被复制的成本多高？"
Round 2: 反身性检验 — "市场共识本身会改变基本面吗？"
Round 3: 尾部风险 — "最坏情况下的下行空间？哪些因素被市场低估？"
```

## 技术产品/创业评估类

```
Round 0: 无约束自由输出 → 基线评估
Round 1: 替代威胁 — "如果这个产品不存在，用户现在怎么解决？切换成本？"
Round 2: 采用曲线 — "从 early adopter 到 mainstream 的鸿沟在哪？"
Round 3: 单位经济学 — "单用户/单次成本与 LTV 的数量级？盈亏平衡点在哪？"
```

## 社会科学/政策类

```
Round 0: 无约束自由输出 → 基线分析
Round 1: 利益相关方 — "各方动机和约束？谁会支持/反对？"
Round 2: 时间动态 — "短期 vs 长期效果？滞后效应？"
Round 3: 反事实 — "如果没有这个政策/事件，会怎样？"
```

## PESTEL 约束链（市场洞察/行业分析/战略规划类）

```
Round 0: 无约束自由输出 → 基线市场判断（市场规模、格局、趋势）
Round 1 (Political):  政策/地缘约束 — "什么政策变化会让这个市场假设完全失效？监管风向是顺风还是逆风？"
Round 2 (Economic):   经济可行性约束 — "收入预期和成本结构之间的缺口多大？谁真正有支付能力和意愿？"
Round 3 (Social):     需求真实性约束 — "实际用户/采购方的动机与行业叙事一致吗？是否存在'伪需求'？"
Round 4 (Technological): 技术就绪度约束 — "TRL实际到哪了？替代技术路径的进步速度是否被低估？窗口期还剩多久？"
Round 5 (Environmental): 环境约束 — "碳成本/ESG要求/环保法规对经济模型的定量冲击？"
Round 6 (Legal):      法律/合规约束 — "合规成本是否被系统性低估？知识产权/数据主权/行业准入的隐性风险？"
Round 7 (Reverse):     PESTEL+ 反向追问 — "如果以上六轮悲观假设全部不成立，最好的结果是什么？哪些被系统性低估的积极因素可能改变结论？⚠️ 此轮为强制执行——PESTEL 天然产生累积悲观偏差。"
```

> **PESTEL 链设计原则：** 前六轮从六个方向攻击同一个气球——才知道它真正的形状。第七轮反向追问是必需的校准——它回答"如果气球本身比你想象的大得多呢？" **PESTEL 是压力测试工具，不是预测工具——它的产出是风险下限，不是期望值。综合结论需对前六轮悲观交集与第七轮乐观上限进行加权。**

## LLM Council 五角色约束链（通用决策/战略判断类）

```
Round 0: 无约束自由输出 → 基线判断
Round 1 (唱反调者):     风险审视 — "这个方案最大的风险和最可能的失败点？列出三个被忽略的脆弱假设。"
Round 2 (第一性原理):   假设重构 — "底层假设是什么？如果全部推翻重建，会得出什么不同的结论？"
Round 3 (扩张主义者):   机会扫描 — "被忽视的积极场景和潜在收益？哪些机会因保守假设被过滤掉了？"
Round 4 (局外人):       盲区检测 — "完全不懂这个行业的人会怎么质疑？类比到另一个行业，这个逻辑成立吗？"
Round 5 (执行者):       落地检验 — "具体怎么落地？第一步行动、关键资源、最常见的执行失败模式？"
```

> **Council 链设计原则：** 五轮不是"五个角度分析"，而是**五种认知偏见矫正器**。唱反调者矫正乐观偏差，第一性原理矫正框架锁定，扩张主义者矫正保守偏差，局外人矫正行业盲区，执行者矫正抽象化倾向。**1个Agent用5轮角色扮演替代5个并行Agent，降低70-80%模型调用成本。**

## 使用方式

每轮追问时将前一轮的结论作为上下文传入，要求模型在新的约束维度下重新审视。**不要只问一轮**——单轮输出倾向走"最可能"路径，隐藏假设不被检验。
