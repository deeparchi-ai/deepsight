# Council 严谨性协议

> Council 的目的不是"达成共识"，是**暴露盲区**。"读起来很一致"不是质量信号——**高一致 + 低多样性 = 共享盲区**，应触发怀疑而非确认。本协议把 Phase 2 从"≥3 位独立提出 + 主席综合"这类靠人头/靠主模型自觉的软规则，改成**可度量、可审计、裁判与球员分离**的硬规则。
>
> 基础设施怎么接（provider/模型/curl 测试）见 [`council-model-setup.md`](council-model-setup.md)；本文件只管**严谨性**，冲突时以本文件为准。

## §1 反同质化：按"模型族"度量，不是按厂商

- **模型族（model family）= 训练谱系不同的底座**：DeepSeek / Moonshot(Kimi) / Qwen / Gemma / GPT-OSS / Nemotron / Llama / GLM … 各算一族。**同族不同尺寸不计入多样性**（共享训练数据 = 共享盲区）。base_url 不同但同族，不算交叉验证。
- **多样性分 D = 审计员覆盖的 distinct 模型族数**（不含主模型）。
- **分歧率下限（disagreement floor）**：盲审第一轮，给每名审计员对**核心结论**记一个立场 ∈ {支持, 质疑, 反对}。计"同向一致度"。
  - 若 **一致度 > 80%（≥4/5 同向）且 D < 3** → 判定**同质化风险**，必须升级处理（二选一），不得直接接受：
    1. 给 ≥2 名审计员**注入更强的对立先验**重跑（例如强制"假设主结论是错的，找出最可能的崩坏点"）；或
    2. **换入一个新的模型族**再审。
  - 高一致 **且** D≥3 才可视为"结论稳健"的弱证据；低一致永远要进 §3 共识聚合。
- 记录 `council_provenance`：每个角色 → {模型族, 模型 id, 是否独立厂商}，写入审计报告，供读者核验审计强度。

## §2 诚实降级阶梯：可信度与多样性绑定

**禁止静默降级**——多样性不足时，必须把影响**落到结论证据级 + 报告显式标注**，让读者看见这次审计有多强。

| D（独立模型族） | 路由 | 对结论证据级的影响 |
|---|---|---|
| ≥3 | ✅ 完整交叉验证 | 审计可支撑 `[A]`（满足"外部独立来源"中的独立性） |
| 2 | ⚠️ 部分 | 结论证据级**封顶 `[B]`**；标注"双族审计·覆盖有限" |
| 1 | ⚠️ 单族 | 封顶 `[B]`，标注"单模型族审计·盲区风险高"；**不得宣称"已跨模型交叉验证"** |
| 0 | ❌ 跳过 | 用增强单模型对抗替代（Phase 1 自审水平），结论显式标"未经跨模型验证" |

## §3 共识的算法定义（不靠人头、不靠感觉）

- 每名审计员输出**结构化批评条目**：`{claim_id, 维度tag, 严重度(1-3), 立场}`。
- **维度 tag 取自固定词表**：`事实性` / `因果链` / `样本与来源` / `反身性` / `尾部风险` / `时间动态` / `利益相关方` / `可证伪性` / `幸存者偏差` / `期望校准`。
- **共识 = 同一 `(claim_id, 维度tag)` 被 ≥2 个*不同模型族*的审计员独立提出**。同族重复**不计入**共识强度（防同族刷票）。
- 输出**共识矩阵**：`claim × 维度 × 命中模型族数`。**共识强度 = 命中的 distinct 模型族数。**

## §4 裁决独立：破"裁判即球员"

主模型既是被审者又当终审 = referee-is-a-player，违背"自洽 ≠ 正确"。规则：

1. **规则优先于裁决**：凡**共识强度 ≥ 2（不同族）**的批评，**自动**触发结论修正 / 证据级下调，**无需主模型"同意"**。
2. **裁决席异族**：当审计员之间互相冲突、需要裁决时，裁决交给**与主模型不同模型族**的一名审计员（或交用户）。主模型不得裁决针对自己结论的分歧。
3. **主模型只能下调，不能上调**：主模型可以采纳批评、下调证据级；**禁止用自己的反驳驳回审计批评以维持原结论或上调证据级**。若主模型确信某批评有误，只能标"主模型存疑"并**保留该批评可见**，不得删除。
4. **终审证据级 = min( Phase 1 自评级, §2 多样性上限, §3 共识修正后级别 )。**

## §5 审计报告必含字段（可审计性）

- `council_provenance`（§1）、`多样性分 D` 与对应 §2 路由
- `首轮一致度` 与是否触发分歧率下限（§1）
- `共识矩阵`（§3）
- `证据级修正表`：每条结论 修正前 → 修正后 + 触发规则
- `审计强度声明`：本结论经几个独立模型族验证、是否封顶、是否"未经跨模型验证"

> 一句话：Council 通过**不是**让多个模型点头，而是让足够**不同**的模型**各自独立**挑出**同一个**毛病——且挑错的权力不在被审的那个模型手里。
